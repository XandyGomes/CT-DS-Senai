Requisito Funcional

- Confirmar reserva -> Permite o usuário concluir a reserva do seu quarto
- Verificar Disponibilidade -> O sistema verifica quais quartos podem ou não serem reservados
- Cadastrar reservas -> Permite o usuário a realizar uma reserva de um quarto
- Gerenciar Serviços Adicionais -> O sistema permite aos funcionários gerenciar diversos serviços

Requisito não Funcional

- Desempenho -> Sistema que dê respostas rápidas para o usuário
- Segurança -> Sistema com dados armazenados com segurança, e protegido contra invasões internas e externas
- Usabilidade -> Sistema facíl de usar, didático e bonito;
- Confiabilidade -> O sistema possui segurança e retorno a todas as ações que realiza

Regra de Negócio

- Política de Check-in e Check-out -> Aplica a obrigação de fazer o check-in e check-out ao cliente
- Classificação por categoria -> O usuário pode buscar quartos pelas categorias que escolher
- Cancelamento com Antecedência -> Permite o usuário possa cancelar sua reserva antecipadamente
- Pagamento antecipado -> Pagamento antecipado para garantir a reserva



Regras de Negócio

-> Aplicar multas em relação ao atraso de devolução ou condições em que o livro foi devolvido
-> Realizar um cadastro na biblioteca para levar os livros
-> Ter a possibilidade de agendar um livro préviamente
-> Prorrogar o praza para a leitura de livros
